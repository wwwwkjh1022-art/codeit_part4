# 제출용 패키지

이 폴더는 제출 기준에 맞춰 필요한 산출물을 한곳에 모아두기 위한 폴더입니다.

## 폴더 구성

- `보고서_원고.md`
  - 제출용 보고서 PDF로 내보내기 위한 원고
- `업무일지_정리.md`
  - README 링크용 업무일지 정리본
- `README_링크_추가용_문구.md`
  - Github Repository README에 붙여 넣을 링크 문구

## 제출 체크리스트

- [ ] Github Repository 링크 제출
- [ ] 보고서 PDF 1개 제출
- [ ] Github Repository README에 보고서 다운로드 링크 첨부
- [ ] Github Repository README에 업무일지 링크 첨부
- [ ] 업무일지가 Notion/블로그 링크가 아니라면 PDF로 변환해 첨부

## 추천 제출 파일명

- 보고서 PDF: `보고서.pdf`
- 업무일지 PDF: `업무일지.pdf`

## 권장 제출 순서

1. `보고서_원고.md`를 PDF로 변환합니다.
2. `업무일지_정리.md`를 PDF로 변환하거나 Notion/블로그 링크로 대체합니다.
3. `README_링크_추가용_문구.md` 내용을 루트 `README.md`에 붙여 넣습니다.
4. 최종 제출 전에 루트 `README.md`에서 링크가 실제로 열리는지 확인합니다.

## README에 채워 넣어야 할 값

- `REPO_URL`: 본인 Github Repository 주소
- `REPORT_URL`: 업로드한 보고서 PDF 주소
- `WORKLOG_URL`: 업무일지 PDF 또는 Notion/블로그 주소

## 현재 프로젝트 기준 메모

- 보고서 원고는 현재 프로젝트 구현 범위를 기준으로 작성해 두었습니다.
- 업무일지 정리본은 기존 개발 일지와 구현 흐름을 제출용으로 다시 정리한 버전입니다.
- 실제 제출 시에는 본인 저장소 URL과 업로드 링크만 바꿔 넣으면 됩니다.
