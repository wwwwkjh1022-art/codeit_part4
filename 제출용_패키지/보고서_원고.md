# 소상공인 AI 홍보 자동화 서비스 보고서

## 1. 프로젝트 개요

본 프로젝트는 소상공인이 상품 또는 서비스 정보만 입력하면 AI가 Instagram, Threads, Blog 채널에 맞는 광고 콘텐츠를 생성하고, 배너 미리보기와 게시 준비 정보까지 함께 제공하는 서비스형 MVP를 구현하는 것을 목표로 했다.

기존의 단순 문구 생성기가 아니라, 실제 제출 과제의 방향에 맞게 "서비스" 형태로 사용할 수 있도록 입력, 생성, 결과 확인, 품질 평가, 이력 관리, 예약 준비 흐름까지 하나의 웹 서비스 안에서 연결했다.

## 2. 문제 정의

소상공인은 좋은 상품과 서비스를 가지고 있어도 온라인 홍보 콘텐츠를 꾸준히 제작하기 어렵다. 특히 Instagram, Threads, Blog는 같은 내용을 그대로 올리기보다 채널 특성에 맞게 문체, 길이, 정보량을 조절해야 한다.

이 프로젝트는 다음 문제를 해결하고자 했다.

- 광고 문구를 만들기 어렵다.
- 채널별로 다른 형식의 글을 각각 작성하기 어렵다.
- 결과물이 실제 게시 전에 쓸 만한지 판단하기 어렵다.
- 향후 자동화까지 확장할 수 있는 구조가 필요하다.

## 3. 목표

- 소상공인 대상 광고 콘텐츠 생성 서비스 구현
- 입력 브리프를 기반으로 3개 채널용 결과 생성
- 광고 문구뿐 아니라 배너 미리보기와 품질 점수 제공
- 생성 결과 저장, 재열람, 재생성, 예약 준비 흐름 지원
- 향후 n8n 기반 자동화로 확장 가능한 API 구조 마련

## 4. 구현 내용

### 4-1. 웹 서비스 구성

- FastAPI 기반 웹 서버 구축
- Jinja2 템플릿 기반 입력 페이지와 결과 페이지 구현
- 사용자가 업종, 상호명, 상품명, 타깃 고객, 홍보 목적, 비주얼 스타일 등을 입력할 수 있도록 설계
- 상품 설명을 비워도 자동 설명이 생성되도록 처리해 입력 부담 완화

### 4-2. AI 광고 콘텐츠 생성

- OpenAI Responses API 기반 채널별 콘텐츠 생성 구조 설계
- Structured Outputs 방식으로 JSON 구조를 강제해 결과 안정성 확보
- OpenAI 호출 실패 시에도 서비스가 멈추지 않도록 mock fallback 생성기 구현

생성 결과는 다음 채널 패키지로 구성했다.

- Instagram: 캡션, 해시태그, ALT 텍스트, visual hook, 추천 게시 시간
- Threads: 본문, reply prompt, short hook, 추천 게시 시간
- Blog: 제목, 도입문, 본문 개요, SEO 키워드, CTA, 메타 설명
- Poster: 보조 포스터용 문구

### 4-3. 배너 미리보기 및 이미지 확장 구조

- Pillow 기반 배너 생성기 구현
- 업로드 이미지가 있으면 이를 활용하고, 없으면 텍스트 중심 광고형 배너를 생성
- Stable Diffusion 3.5, OpenAI Image, Gemini 계열 이미지 생성 provider를 확장 가능한 구조로 설계
- 실제 이미지 API를 쓰지 않아도 발표와 시연이 가능하도록 프롬프트 전용 및 로컬 fallback 흐름 제공

### 4-4. 품질 평가 및 검수 지원

- 생성 결과의 품질을 정량적으로 보기 위한 평가 기준 설계
- 후킹, 구체성, 채널 적합성, CTA, 과장광고 위험, 구조 완성도를 기준으로 평가
- 결과 화면에서 전체 점수, 채널별 점수, 수정 제안, 재생성 힌트를 함께 제공
- 자동 예약 전 체크리스트를 제공해 사용자가 게시 전에 검수할 수 있도록 구성

### 4-5. 이력 관리 및 자동화 준비

- 생성 결과를 로컬 JSON 저장소에 저장
- `/campaigns` 페이지에서 과거 생성 결과를 다시 확인 가능
- 같은 브리프로 재생성 가능
- 예약 준비 상태와 게시 작업 상태 관리 가능
- n8n이 사용할 수 있는 JSON API를 별도로 구현해 자동화 연동 구조 마련

## 5. 기술 스택

- Backend: Python 3.12, FastAPI, Uvicorn
- Schema/Config: Pydantic, Pydantic Settings
- Frontend: HTML, CSS, Jinja2, Vanilla JavaScript
- AI: OpenAI Responses API, mock generator
- Image: Pillow
- Test: pytest, pytest-asyncio

## 6. 핵심 설계 의사결정

### 6-1. Streamlit 대신 FastAPI 선택

과제 제출 수준을 넘어서 서비스 구조를 보여주기 위해 FastAPI + HTML 기반 구조를 선택했다. 이 방식은 추후 OAuth, 예약 게시 API, 자동화 API, 상태 저장 구조와 자연스럽게 연결할 수 있다는 장점이 있었다.

### 6-2. 단일 문구 생성기가 아닌 채널 패키지 구조 선택

같은 광고 목적이라도 Instagram, Threads, Blog는 필요한 문체와 정보 구조가 다르기 때문에, 단일 캡션 결과가 아니라 채널별 패키지 구조로 확장했다.

### 6-3. 서비스 안정성을 위한 fallback 설계

외부 API 오류가 발생해도 결과 화면까지는 도달할 수 있도록 fallback 생성기를 넣었다. 이를 통해 발표나 시연 중 API 이슈가 생겨도 서비스가 완전히 멈추지 않도록 했다.

## 7. 테스트 및 평가 결과

프로젝트에는 기능 테스트와 품질 평가 테스트를 함께 구성했다.

- 생성 품질 평가 리포트 기준 4개 시나리오 모두 PASS
- 최신 평가 리포트 기준 overall score 99, channel score 100 수준의 결과 확인
- 품질 리포트와 자동 검수 기준을 별도 리소스로 정리

평가 리포트 요약:

- provider: `mock-eval`
- passed: `4/4`
- 평가 항목: Hook, Specificity, Channel Differentiation, CTA, Compliance, Structure 등

## 8. 성과

- 단순 실습 코드가 아니라 실제 웹 서비스 형태의 MVP를 구현했다.
- 소상공인 광고 콘텐츠 생성이라는 과제 주제에 직접적으로 맞는 결과물을 만들었다.
- 3채널 패키지 생성, 배너 미리보기, 품질 평가, 이력 관리, 자동화 준비 API까지 연결해 서비스 완성도를 높였다.
- 문서화와 평가 자료를 함께 준비해 제출과 발표 대응력을 확보했다.

## 9. 한계

- 실제 Instagram/Threads/Blog 게시 API는 완전 연동하지 않았고 mock 기반 흐름으로 설계했다.
- 이미지 생성은 비용과 환경 제약을 고려해 실제 API 호출보다 fallback 중심으로 구성했다.
- 일부 TestClient 기반 테스트는 현재 환경에서 재현성 이슈가 있어 검증 과정에 한계가 있었다.

## 10. 개선 방향

- 실제 SNS/블로그 게시 API 연동
- DB 기반 저장 구조로 전환
- 사용자별 캠페인 관리 기능 추가
- 이미지 생성 품질 고도화
- 프롬프트/평가 기준 고도화
- 관리자 검수 및 승인 워크플로우 추가

## 11. 결론

본 프로젝트는 "소상공인을 위한 광고 콘텐츠 생성 서비스"라는 과제 목표에 맞춰 실제 사용 흐름을 고려한 MVP를 구현한 결과물이다. 입력부터 채널별 콘텐츠 생성, 배너 미리보기, 품질 평가, 이력 관리, 자동화 확장 기반까지 연결했다는 점에서 제출용 프로젝트로 충분한 완성도를 갖췄다.
